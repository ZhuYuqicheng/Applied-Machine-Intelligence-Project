Weather

We observed that photovoltaics account for a large proportion of total generation. Weather, especially radiation, has an influence on the generation of photovoltaics. Hence, introducing weather-related features, such as air temperature at 200cm high and 20cm high, temperature at 5cm and 20cm away from the ground, humidity, wind speed and rainfall, into our research is necessary. It should be noted that the weather features are collected from Leipzig.

History spot price

History spot price in Leipzig is the most correlated feature to predict the future spot price, which can be obtained from Montel API through corresponding token and correct indexing.

Electricity consumption

we collect the data of total power and residual power consumption for electricity consumption, electricity power generated by wind onshore, fossil brown and hard coal, hydro pumped storage and photovoltaics, from the website smard.de.

To improve the predictive accuracy of ARIMA model, we also need to do feature engineering which includes the feature extraction, feature selection and interpretation. For feature extraction we use a scrolling window with a size of 24 to divide the spot price into many short sequences firstly, then we used the Python package "tsfresh" to extract 786 features in each window. After that, we select the features in two stages, respectively. In the first stage, we used Kendall rank correlation coefficients with a significance level of 0.05 to filter out uncorrelated variables. After this, we continued to use Lasso regression, keeping only the features with non-zero coefficients. At last, we just used the top 17 features.

Workflow

  • 2021

  • Feature collection

    Through literature review and group discussion, we initially summarized the features that may influence electricity prices and collected those features from the corresponding sources, such as Montel API, smard.de and so on.

    20 APR 2021

  • Data preprocessing

    There are a lot of works in the data preprocessing. We merge these data collected from different sources into one file and downsample or upsample them in hours. We also need to deal with missing values by using the way such as auto-regression model in the spot electricity price dataset. For ARIMA model, we used scrolling window technique and tsfresh module in python to extract the features. Besides that, we selected features by using Kendall rank correlation coefficients with a significance level of 0.05 and Lasso regression method.

    8 JUN 2021
    • {#
  • 2015
    • #}

  • Model training

    There are two training processes involved in the ARIMA model. During the training process for the Lasso regression, we gradually increased the value of alpha and found that the performance of our model is almost the same until the alpha equals 2. In order to avoid overfitting and shorten the prediction time, we set the value of alpha equals 2 at the end. The length of training set can also significantly influence the performance of ARIMA model. We use sMAPE (symmetric mean absolute percentage error) as a performance metric and found that the performance of the ARIMA model with the training length of 2 months is the best.

    25 JUN 2021

  • Model evaluation

    To provide statistically robust evaluation of ARIMA model, we used time-based cross validation, which forms a type of “sliding window” training approach. This method slides a window over the training samples while using several future samples as a test set.

    8 AUG 2021

  • Integration to the Web-Interface

    Three results of predictions using these two models for the price 1 hour ahead, 1 day ahead and 1 week ahead will be shown on this web interface. Users can click on the prediction buttons for each of the two models to observe the results.

    20 AUG 2021
